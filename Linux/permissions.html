File Permissions & Ownership

In Linux, everything is a file. Understanding who can read, write, or execute those files is critical for security.

Understanding the String

When you run ls -l, you see a 10-character string like this:

- rwx r-x r--
● User: Owner of the file.
● Group: Users who are members of the file's group.
● Others: Everyone else on the system.
chmod (Change Mode)

Changes the access permissions of a file. You can use numbers (755) or symbols (u+x).

No.PermissionValue
4Read (r)Read Only
2Write (w)Edit/Delete
1Execute (x)Run as Script
chmod 755 script.sh

755 = User(4+2+1), Group(4+0+1), Other(4+0+1)

chown (Change Owner)

Changes which user and group "own" the file. This is vital when deploying web servers like Nginx or Apache.

sudo chown www-data:www-data index.html

Format: user:group. Usually requires sudo.

umask

Sets the default permissions for newly created files and directories. It acts as a "filter" that subtracts from the maximum possible permission.

umask 022
